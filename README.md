# 손글씨 숫자 분류기

손으로 쓴 숫자(0~9) 이미지를 분류하는 머신러닝/딥러닝 모델 모음입니다.
같은 데이터셋(scikit-learn digits)을 KNN과 신경망 두 가지 방식으로 각각 풀어보고 비교합니다.

## 사용 기술
- Python
- scikit-learn (KNN, 교차 검증, 혼동 행렬)
- PyTorch (신경망, 역전파, Adam)

---

## 1. KNN 분류기

### 작동 방식

1. **데이터 로드** — sklearn에서 손글씨 숫자 데이터를 가져옵니다.
2. **분리** — train/test로 나눕니다 (test 비율 0.2).
3. **학습** — 학습 데이터로 KNN을 학습시킵니다.
4. **평가** — 테스트 데이터로 정확도를 측정하고, 교차 검증으로
   안정적인 성능을 확인합니다.
5. **분석** — 혼동 행렬로 어떤 숫자를 어떤 숫자와 헷갈리는지 확인합니다.

### 실행 방법
1. `pip install scikit-learn`
2. `python digits.py`   (실제 파일명에 맞게)

### 배운 점

- **KNN 작동 방식**: 데이터를 저장만 해두고(게으른 학습), 예측할 때
  새 데이터와 모든 학습 데이터의 유클리드 거리를 계산해 가장 가까운
  이웃들을 찾아 다수결로 정한다.
- **하이퍼파라미터 선택**: 정확도는 n_neighbors=7이 높았지만 한 번의
  분할 운일 수 있어, 교차 검증 평균이 높고 안정적인 3을 선택했다.
- **모든 개선이 통하는 건 아니다**: weights='distance'를 시도했으나
  이 데이터에서는 효과가 없어 uniform으로 되돌렸다.
- **혼동 행렬 분석**: 모델이 9를 3, 4, 5와 주로 헷갈린다는 것을
  확인했다. 실수가 대부분 모양이 비슷한 숫자끼리 일어났다.

## 2. 신경망 분류기 (PyTorch)

KNN은 데이터를 저장만 할 뿐 "학습"이라 부르기 애매했다. 이번엔 진짜로
가중치가 반복적으로 스스로를 수정해나가는 과정을 직접 만들어봤다.

### 기본 모델 구조

입력(64, 픽셀 수) → 숨은 층(32, ReLU) → 출력(10, 클래스 수)

### 작동 방식

1. **데이터 준비** — numpy 배열을 PyTorch 텐서로 변환하고, train/test로 분리.
2. **배치 구성** — DataLoader로 32장씩 랜덤하게 묶어서 공급.
3. **순전파** — 이미지를 모델에 통과시켜 10개 클래스 점수(로짓)를 얻음.
4. **loss 계산** — Softmax + CrossEntropyLoss로 "얼마나 틀렸는지"를 숫자 하나로 계산.
5. **역전파** — `loss.backward()`로 모든 가중치 각각에 대한 기울기를 자동 계산.
6. **가중치 수정** — Adam 옵티마이저가 기울기를 이용해 가중치를 한 걸음씩 수정.
7. 3~6을 배치마다 반복하고(1 에포크), 이를 20 에포크 반복.
8. **평가** — 한 번도 학습에 쓰지 않은 test 데이터로 최종 정확도 측정.

### 실행 방법
1. `pip install torch`
2. `python neural_network.py`   (실제 파일명에 맞게)

### 기본 모델 결과
- 20 에포크 학습 후 평균 loss: 2.03 → 0.048
- 테스트 정확도: 97.22%

### 구조 비교 실험

같은 데이터로 여러 구조를 각각 20번씩 반복 학습해서 평균/표준편차로 비교했다
(1번 실행 결과만으로는 랜덤 초기화에 따라 순위가 계속 뒤집혀서, 여러 번
반복해 평균 내는 게 KNN의 교차 검증과 같은 이유로 꼭 필요했다).

| 구조 | 평균 정확도 | 표준편차 |
|---|---|---|
| 1개 층 (64→32→10) | 97.36% | 0.40% |
| 2개 층, 점진적 축소 (64→32→16→10) | 96.94% | 0.49% |
| 2개 층, 폭 유지 (64→32→32→10) | 96.94% | 0.34% |
| 1개 층 + Leaky ReLU | 97.31% | 0.60% |
| 2개 층 축소 + Leaky ReLU | 96.76% | 0.52% |
| 2개 층 유지 + Leaky ReLU | 97.06% | 0.51% |

**결론**: 이 문제(digits, 1437장, 10클래스)에서는 층을 늘리거나 Leaky ReLU로
바꿔도 뚜렷한 성능 향상이 없었다. 오히려 1개 층 구조가 가장 단순하면서도
꾸준히 가장 높은 평균을 보였다. 차이 자체는 표준편차보다 작아 통계적으로
확정적이진 않지만, "모델을 복잡하게 만들수록 좋아진다"는 가정이 항상 맞는
건 아니라는 걸 직접 확인했다. digits 데이터가 비교적 단순한 문제라서,
32개 뉴런짜리 층 하나로 이미 충분한 표현력을 갖고 있었던 것으로 보인다.

### 혼동 행렬 비교 (KNN vs 신경망)

같은 test 데이터로 신경망(1개 층 모델)의 혼동 행렬을 확인해봤다.

- KNN: 9를 3, 4, 5와 반복적으로 헷갈리는, 한 숫자에 실수가 집중되는 패턴.
- 신경망: 실수가 여러 숫자쌍(3→5, 5→6, 5→9, 6→0, 7→9, 8→1, 8→5, 9→8)에
  걸쳐 골고루 분산되어 있고, 한 곳에 몰리는 패턴은 보이지 않았다.

두 모델이 "비슷함"을 판단하는 기준 자체가 다르기 때문으로 보인다 — KNN은
픽셀 단위 거리로 생김새가 비슷한 것끼리 헷갈리는 반면, 신경망은 학습된
추상적 특징으로 구분해서 KNN과는 다른 지점에서 실수한다.

### 배운 점

- **KNN과의 근본적 차이**: KNN은 데이터를 저장만 하지만, 신경망은
  랜덤한 가중치에서 시작해 예측→오차 계산→기울기 계산→수정을
  반복하며 실제로 값이 바뀌어간다. `loss.backward()` 전후로 가중치를
  직접 찍어보고 나서야 이 차이가 체감됐다.
- **비선형(ReLU)이 필요한 이유**: 활성화 함수 없이 층을 아무리 쌓아도
  수학적으로 직선 하나와 동일해진다. XOR처럼 직선 하나로 못 가르는
  문제를 직접 만들어 검증해봄으로써 왜 층 사이에 ReLU가 꼭 필요한지
  이해했다.
- **"가중치가 안 바뀐다"는 착각 잡아내기**: 특정 자리의 가중치가
  그대로인 걸 보고 버그라 생각했는데, 알고 보니 (1) 입력 픽셀 값이
  0이라 기울기가 0인 경우와 (2) ReLU가 뉴런을 죽여서 기울기가 0인
  경우, 두 가지 다른 원인이 섞여 있었다. 직접 실험으로 둘을 구분해냈다.
- **미니배치의 의미**: 32장의 평균 loss로 학습시키는 이유는 이미지
  한 장의 우연한 특성에 휘둘리지 않고, 그 배치 전체에 평균적으로
  가장 도움이 되는 방향을 찾기 위해서다. 개별 이미지에서는 죽어있던
  기울기가, 배치 평균에서는 다른 이미지들 덕분에 살아나기도 한다.
- **옵티마이저 (SGD → SGD+Momentum → Adam)**: SGD는 매번 그 순간의
  기울기만 보고 움직여 지그재그로 헤맨다. Momentum은 관성(최근 방향의
  지수이동평균)을 더해 이를 완화한다. Adam은 여기에 파라미터별
  맞춤 보폭(기울기 변동성 기반)까지 더해, 자주 흔들리는 가중치는
  조심스럽게, 안정적인 가중치는 과감하게 움직이도록 자동 조절한다.
- **직접 유도해서 검증**: Softmax+CrossEntropy의 기울기가
  "예측확률 - 정답표시(0 or 1)"로 깔끔하게 떨어진다는 걸 로그/지수
  미분으로 직접 유도해봤다. 정답 클래스는 항상 끌어올리고 오답은
  끌어내리는 방향이 나오며, 전체 기울기의 합이 0이 되는 것도 확인했다
  (확률이라는 한정된 자원의 재분배).
- **랜덤 초기화의 영향력**: 같은 구조를 한 번만 학습시키면 결과가 운에 따라
  꽤 흔들린다는 걸 직접 겪었다 (5번 반복했을 때도 두 구조의 순위가 실행마다
  뒤바뀜). 20번 반복하고 나서야 안정적인 비교가 가능했다.
- **복잡한 모델이 항상 이기는 건 아니다**: 파라미터가 더 많은 2개 층
  구조가 오히려 살짝 낮은 평균을 보였다. 데이터 양과 문제 난이도에 맞는
  적절한 복잡도가 있다는 걸 숫자로 확인했다.
- **이론적 개선이 항상 실전에서 통하지 않는다**: Leaky ReLU가 "죽은 뉴런
  문제"를 이론적으로 완화한다는 걸 알고 있었지만, 실제로는 기본 ReLU와
  거의 차이가 없었다. KNN에서 weights='distance'가 안 통했던 것과 같은
  종류의 교훈.
- **디버깅 두 번**: `self.hidden`을 두 층에 똑같이 써서 한 층이 통째로
  사라진 것과, 결과 변수 이름을 겹쳐 써서 값이 덮어써진 것. 둘 다 "같은
  이름을 재사용하면 이전 것이 조용히 사라진다"는 같은 원리에서 나온
  실수였다.
